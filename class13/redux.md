# Redux vs Redux Toolkit

## Setup

**Redux (vanilla)**
```js
import { createStore, combineReducers, applyMiddleware } from 'redux';
import thunk from 'redux-thunk';

const store = createStore(
  combineReducers({ cart: cartReducer, products: productsReducer }),
  applyMiddleware(thunk)
);
```

**Redux Toolkit**
```js
import { configureStore } from '@reduxjs/toolkit';

const store = configureStore({
  reducer: { cart: cartReducer, products: productsReducer },
});
// thunk middleware included by default
```

## Reducers

**Redux (vanilla)** — manual immutability, verbose switch statements
```js
const initialState = { items: [], loading: false };

function productsReducer(state = initialState, action) {
  switch (action.type) {
    case 'LOAD_PRODUCTS_PENDING':
      return { ...state, loading: true };
    case 'LOAD_PRODUCTS_FULFILLED':
      return { ...state, loading: false, items: action.payload };
    default:
      return state;
  }
}
```

**Redux Toolkit** — uses Immer under the hood, mutate directly
```js
const productsSlice = createSlice({
  name: 'products',
  initialState: { items: [], loading: false },
  reducers: {},
  extraReducers: (builder) => {
    builder
      .addCase(loadProducts.pending, (state) => { state.loading = true; })
      .addCase(loadProducts.fulfilled, (state, action) => {
        state.loading = false;
        state.items = action.payload; // direct mutation is safe here
      });
  },
});
```

## Action Creators

**Redux (vanilla)** — define types and creators manually
```js
const LOAD_PRODUCTS = 'LOAD_PRODUCTS';

const loadProducts = () => async (dispatch) => {
  dispatch({ type: 'LOAD_PRODUCTS_PENDING' });
  try {
    const data = await fetchProducts();
    dispatch({ type: 'LOAD_PRODUCTS_FULFILLED', payload: data });
  } catch (err) {
    dispatch({ type: 'LOAD_PRODUCTS_REJECTED', payload: err.message });
  }
};
```

**Redux Toolkit** — `createAsyncThunk` handles pending/fulfilled/rejected automatically
```js
const loadProducts = createAsyncThunk('products/loadProducts', async () => {
  return await fetchProducts();
});
```

## Immutability

| | Redux (vanilla) | Redux Toolkit |
|---|---|---|
| How | Spread operators manually | Immer (mutate directly) |
| Risk | Easy to accidentally mutate state | Safe by default |

---

## Boilerplate Comparison

| Feature | Redux | Redux Toolkit |
|---|---|---|
| Action types | Manual constants | Auto-generated |
| Action creators | Manual functions | Auto-generated by `createSlice` |
| Async actions | Manual thunk + 3 action types | `createAsyncThunk` |
| Immutability | Manual spreads | Immer built-in |
| DevTools | Manual setup | Included by default |
| Middleware | Manual `applyMiddleware` | Pre-configured |

---

## When to use which

- **Redux Toolkit** — always, for any new project. It's the official recommended approach.
- **Vanilla Redux** — only if maintaining a legacy codebase that hasn't migrated yet.
